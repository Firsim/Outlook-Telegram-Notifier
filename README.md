# 🌟 Outlook-Telegram-Notifier
Программа для мониторинга входящих писем в Microsoft Outlook и отправки уведомлений о новых письмах в Telegram.

### ⚠️ Внимание!
Для работы программы вам необходимо создать бота в [BotFather](https://t.me/BotFather).

А далее через бота можно присылать себе уведомления в группу или личным сообщением.
Для того, что бы узнать ID чата группы или личных сообщений используйте бота: [Get My ID](https://t.me/getmyid_bot).

### 📊 Мониторинг работы программы

Программа использует сторонние библиотеки для взаимодействия с API Microsoft и API Outlook. Это может привести к периодическим зависаниям или аварийным завершениям работы. Однако последняя версия программы демонстрирует улучшенную стабильность: в течение месяца тестирования не было зафиксировано ни одного сбоя.

Для обеспечения мониторинга состояния программы и предотвращения запуска дублирующих экземпляров встроена поддержка WEB-сервера. Этот сервер предоставляет информацию о состоянии службы и позволяет интегрировать программу с внешними системами мониторинга.

#### Настройка и использование WEB-сервера

1. **Запуск программы с параметрами WEB-сервера**  
   При запуске программы необходимо указать IP-адрес и порт, на которых будет работать WEB-сервер. Например:
   ```json
   {
     "ip": "0.0.0.0",
     "port": 9999
   }
   ```

2. **Проверка состояния службы**  
   После запуска программы вы можете проверить её состояние, отправив HTTP-запрос на указанный адрес. Например, откройте браузер и перейдите по пути:
   ```
   http://127.0.0.1:9999/
   ```
   В ответ вы получите JSON-объект, содержащий информацию о состоянии службы:
   ```json
   {
     "service": "OTN",
     "status": "UP"
   }
   ```
   - `"service"` — название службы (в данном случае `OTN`).
   - `"status"` — текущее состояние (`"UP"` означает, что служба активна).

3. **Интеграция с системами мониторинга**  
   Для дополнительной диагностики и мониторинга через сторонние приложения (например, Zabbix) необходимо:
   - Убедиться, что указанный порт открыт для входящих TCP-соединений.
   - Настроить приложение для проверки доступности WEB-сервера по указанному IP-адресу и порту.

   Пример настройки в Zabbix:
   - Создайте элемент данных (item) с типом проверки `Simple check`.
   - Укажите IP-адрес и порт (например, `127.0.0.1:9999`).
   - Настройте триггеры для оповещения о недоступности службы.

#### Рекомендации по использованию

- Убедитесь, что WEB-сервер запущен на уникальном порту, чтобы избежать конфликтов с другими приложениями.
- Если программа используется в локальной сети, убедитесь, что порт доступен только для доверенных устройств.
- Для повышения безопасности рекомендуется ограничить доступ к WEB-серверу с помощью файрвола или других средств защиты.

- Автоматический запуск Outlook, если приложение не запущено

## ⚡ Конфигурация
Создайте файл config.json в директории с программой (`otn.exe`):
```
{
  "telegram": {
    "bot_token": "123456:ABCdefGhIJKlmNoPQRstuVWXyz",
    "default_chat_id": "-1001234567890",
    "use_emojis": true
  },
  "check_interval_seconds": 60,
  "logging_enabled": true,
  "file_logging_enabled": false,
  "start_minimized": false,
  "cut_text": "From:",
  "folders": [
    {
      "name": "Входящие",
      "chat_id": "-1001234567891",
      "message_length": 0
    },
    {
      "name": "Zabbix",
      "chat_id": "-1001234567892",
      "message_length": 500
    }
  ]
,
  "ip": "127.0.0.1",
  "port": 9999
}
```

### 📜 Параметры конфигурации:
telegram:
 - bot_token: Токен вашего Telegram бота.
 - default_chat_id: ID чата по умолчанию, если не указан chat_id для folders, будет отправляться в этот чат.
 - use_emojis: Использовать эмодзи оформление в сообщениях.

check_interval_seconds: Интервал проверки новых сообщений (в секундах).  
logging_enabled: Включение/выключение логирования операций программы в окне программы.  
file_logging_enabled: Включение/выключение логирования операций программы в файл `otn.log`.  
start_minimized: Запуск программы в свернутом состояние, будет отображен значок в области уведомлений (системный трей).  
cut_text: текст, который используется для обрезания длинных сообщений. Может быть пустым (`""`), если обрезание текста не требуется.  
  
folders: Массив отслеживаемых папок:
 - name: Имя папки в Outlook (лучше скопировать на прямую из Outlook).
 - chat_id: ID чата для уведомлений (может быть 0, тогда сообщение будет отправлено в чат default_chat_id).
 - message_length: 
	0 - не отправлять текст письма.
	N - отправлять первые N символов.

ip: IP адрес 127.0.0.1 или 0.0.0.0 для WEB-сервера, который нужен для диагностики работы приложения  
port: порт на котором работает WEB-сервер для диагностики

## 📝 Логирование

Программа записывает логи в файл `otn.log`, который создается в той же директории, где находится исполняемый файл. Если включено логирование (`file_logging_enabled: true`), все события будут записаны в этот файл. Может пригодится, если по какой то причине программа не запускается и не показывает стартовое окно.

#### Пример лога:

```
2025/02/17 11:41:58 Ошибка отправки в Telegram: неверный статус код: 400, тело ответа: {"ok":false,"error_code":400,"description":"Bad Request: can't parse entities: Unsupported start tag \"mail@example.com\" at byte offset 129"}
```
## ⏸️ Зависания
Если программа зависла вы можете завершить процесс через диспетчер устройсв или воспользуйтесь файлом `kill_otn.bat`

## Пояснение к полям конфигурации

#### 1. **`telegram.bot_token`**
- **Описание**: Токен вашего Telegram-бота.
- **Формат**: Строка вида `123456:ABCdefGhIJKlmNoPQRstuVWXyz`.
- **Как получить**:
  - Создайте бота через [BotFather](https://t.me/BotFather).
  - После создания бота BotFather выдаст вам токен.
- **Пример**: `"123456789:ABCdefGhIJKlmNoPQRstuVWXyz"`

---

#### 2. **`telegram.default_chat_id`**
- **Описание**: Идентификатор чата или канала, куда будут отправляться уведомления по умолчанию.
- **Формат**:
  - Число (например, `-1001234567890`) — для приватных чатов или каналов.
  - Строка (например, `"@channel_name"`) — для публичных каналов.
- **Как получить**:
  - Для числового идентификатора используйте бота [Get My ID](https://t.me/getmyid_bot).
  - Для публичных каналов используйте имя канала с символом `@`.
- **Пример**: `"-1001234567890"` или `"@my_channel"`

---

#### 3. **`telegram.use_emojis`**
- **Описание**: Включение/выключение использования эмодзи в сообщениях.
- **Значения**:
  - `true` — использовать эмодзи.
  - `false` — не использовать эмодзи.
- **Пример**: `true`

---

#### 4. **`check_interval_seconds`**
- **Описание**: Интервал проверки новых писем в Outlook (в секундах).
- **Диапазон значений**: От `10` до `1000`.
- **Пример**: `60` (проверка каждую минуту).

---

#### 5. **`logging_enabled`**
- **Описание**: Включение/выключение логирования операций программы.
- **Значения**:
  - `true` — включить логирование.
  - `false` — отключить логирование.
- **Примечание**: Логи отображаются в окне программы. Если установить параметр `false`, то логи в окно программы выводится не будут. Максимально возможно 1000 лог сообщений.
- **Пример**: `true`

---

#### 6. **`file_logging_enabled`**
- **Описание**: Включение/выключение логирования операций программы в файл.
- **Значения**:
  - `true` — включить логирование.
  - `false` — отключить логирование.
- **Примечание**: Логи записываются в файл `otn.log` в той же директории, где находится программа. Программа не чистит лог файл самостоятельно, используйте данную опцию, если программа не запускается или другие ошибки.
- **Пример**: `true`

---

#### 7. **`cut_text`**
- **Описание**: Текст, который используется для обрезания длинных сообщений.
- **Формат**: Строка.
- **Требования**:
  - Минимальная длина: `4 символа`.
  - Может быть пустым (`""`), если обрезание текста не требуется.
- **Пример**: `"STOP"`

---

#### 8. **`folders`**
- **Описание**: Список папок Outlook, которые программа будет мониторить.
- **Формат**: Массив объектов, каждый из которых описывает одну папку.

##### Поля объекта `folders`:
- **`name`**:
  - **Описание**: Название папки в Outlook.
  - **Формат**: Строка.
  - **Максимальная длина**: `150 символов`.
  - **Пример**: `"Inbox"` или `"Входящие"`

- **`chat_id`**:
  - **Описание**: Идентификатор чата или канала, куда будут отправляться уведомления для этой папки.
  - **Формат**:
    - Число (например, `-1001234567890`) — для приватных чатов или каналов.
    - Строка (например, `"@channel_name"`) — для публичных каналов.
    - Может быть `0`, тогла сообщения будут отправляться в `default_chat_id`
  - **Пример**: `"-1000987654321"` или `"@urgent_channel"`

- **`message_length`**:
  - **Описание**: Максимальная длина сообщения, которое может быть отправлено для этой папки.
  - **Диапазон значений**: От `0` до `4000`. Где `0`, что сообщение из email не будет присылаться в уведомление.
  - **Пример**: `500`

---

#### 9. **`ip`**
- **Описание**: IP адрес для WEB-сервера, который можно использовать для мониторинга работы программы. Если указывать IP адрес `"127.0.0.1"`, то программа будет висеть только на `localhost`. Так же этот IP адрес + Port используется для предотварещния запуска дупбликата приложения.
- **Формат**: Строка.
- **Требования**:
  - IP адрес  `"127.0.0.1"` или  `"0.0.0.0"`.
- **Пример**: `"127.0.0.1"`
---

#### 10. **`port`**
- **Описание**: Номер порта на котором будет работать WEB-сервер для мониторинга.
- **Формат**: Число.
- **Требования**:
  - Минимальная длина: `4 символа`.
  - Должен быть в диапазоне от 1024 до 49151.
- **Пример**: `"9999"`

## 🌐 Поддержка прокси-серверов

Программа поддерживает работу через **HTTP/HTTPS** и **SOCKS5** прокси-серверы с авторизацией. Это позволяет использовать приложение в корпоративных сетях, за фаерволами или для повышения приватности соединений с Telegram API.

### ⚙️ Настройка прокси в `config.json`

Добавьте блок `"proxy"` в корень конфигурационного файла:

```json
{
  "telegram": {
    "bot_token": "123456:ABCdefGhIJKlmNoPQRstuVWXyz",
    "default_chat_id": "-1001234567890",
    "use_emojis": true
  },
  "proxy": {
    "enabled": true,
    "type": "http",
    "host": "proxy.example.com",
    "port": 8080,
    "username": "myuser",
    "password": "mypass",
    "no_proxy": ["localhost", "127.0.0.1"]
  },
  "check_interval_seconds": 60,
  "logging_enabled": true,
  "file_logging_enabled": false,
  "start_minimized": false,
  "cut_text": "From:",
  "folders": [
    {
      "name": "Входящие",
      "chat_id": "-1001234567891",
      "message_length": 0
    }
  ],
  "ip": "127.0.0.1",
  "port": 9999
}
```

### 📜 Параметры блока `proxy`:

| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `enabled` | `boolean` | ✅ | Включить (`true`) или выключить (`false`) использование прокси |
| `type` | `string` | ✅ | Тип прокси: `"http"`, `"https"` или `"socks5"` |
| `host` | `string` | ✅ | IP-адрес или домен прокси-сервера |
| `port` | `integer` | ✅ | Порт прокси-сервера (1–65535) |
| `username` | `string` | ❌ | Логин для авторизации на прокси (если требуется) |
| `password` | `string` | ❌ | Пароль для авторизации на прокси (если требуется) |
| `no_proxy` | `array` | ❌ | Список хостов, которые будут обходить прокси (например, локальные адреса) |

> 💡 **Важно**: Даже если ваш прокси-сервер поддерживает HTTPS-соединение, в поле `type` указывайте `"http"` — это определяет **транспортный протокол** для подключения к самому прокси, а не к целевому сайту.

### 🧪 Примеры конфигурации

#### 🔹 Прокси без авторизации
```json
"proxy": {
  "enabled": true,
  "type": "http",
  "host": "192.168.1.100",
  "port": 3128
}
```

#### 🔹 SOCKS5 прокси с авторизацией
```json

  "enabled": true,
  "type": "socks5",
  "host": "socks.example.com",
  "port": 1080,
  "username": "user123",
  "password": "pass456"
}
```

#### 🔹 Прокси с обходом локальных адресов
```json

  "enabled": true,
  "type": "http",
  "host": "proxy.corp.local",
  "port": 8080,
  "username": "domain\\user",
  "password": "secret",
  "no_proxy": ["localhost", "127.0.0.1", "192.168.0.0/16"]
}
```

#### 🔹 Отключить прокси (по умолчанию)
```json
"proxy": {
  "enabled": false
}
```

### 🔍 Диагностика и логирование

При включённом логировании (`logging_enabled: true`) программа выводит статус инициализации прокси:

```
[2026/04/13 12:00:00] Прокси включён: http://proxy.example.com:8080 (с авторизацией)
[2026/04/13 12:00:00] HTTP-прокси настроен: proxy.example.com:8080
[2026/04/13 12:00:00] HTTP-клиент инициализирован (таймаут: 30с)
```

Если прокси выключен:
```
[2026/04/13 12:00:00] Прокси выключен: используется прямое подключение
```

При ошибке подключения:
```
[2026/04/13 12:00:00] Ошибка настройки HTTP-прокси: proxy: auth required
```

### 🛠️ Частые проблемы и решения

| Проблема | Возможная причина | Решение |
|----------|------------------|---------|
| `proxy: unknown scheme: http` | Используется `proxy.FromURL()` вместо `http.ProxyURL` | Обновите программу до версии ≥ 1.06 |
| `connection refused` | Прокси-сервер недоступен или неверный порт | Проверьте `host:port`, доступность сервера, фаервол |
| `proxy: auth required` | Прокси требует авторизацию, но логин/пароль не указаны | Добавьте `username` и `password` в конфиг |
| `dial tcp: lookup proxy.example.com: no such host` | DNS не разрешает имя прокси | Используйте IP-адрес или проверьте DNS-настройки |
| Программа не отправляет уведомления, но не падает | Прокси блокирует запросы к `api.telegram.org` | Добавьте `api.telegram.org` в `no_proxy` или проверьте правила прокси |

### 🔄 Как протестировать прокси

1. Включите логирование: `"logging_enabled": true`
2. Запустите программу: `otn.exe`
3. Проверьте вывод: должны появиться строки `🔌 Прокси включён...` и `✅ ... настроен`
4. Отправьте тестовое письмо в отслеживаемую папку Outlook
5. Убедитесь, что уведомление пришло в Telegram

Если уведомления не приходят — проверьте логи на наличие ошибок и сверьтесь с таблицей выше.

---

> 💡 **Совет**: Для отладки можно временно отключить прокси (`"enabled": false`), чтобы убедиться, что проблема именно в настройках прокси, а не в токене бота или `chat_id`.
